# YANG Model Designer: инструкция пользователя

Этот файл нужно обновлять при каждом добавлении или изменении пользовательского функционала.

## Назначение приложения

YANG Model Designer - desktop-приложение для работы с YANG-моделями.

Текущее состояние приложения: MVP с частичной поддержкой YANG 1.1 RFC 7950.

Приложение позволяет:

1. Создавать новую YANG-модель из шаблона.
2. Открывать `.yang` файлы.
3. Просматривать структуру модели в виде дерева узлов.
4. Просматривать свойства выбранного узла.
5. Редактировать YANG-текст вручную.
6. Редактировать часть свойств выбранного узла через графический интерфейс.
7. Добавлять дочерние YANG-узлы через выбор типа.
8. Удалять выбранный узел.
9. Сохранять модель в файл.
10. Запускать базовую проверку модели.
11. Сохранять неизвестные и vendor-specific YANG statements при обратной генерации текста.
12. Различать Read-Write и Read-Only узлы в дереве.
13. Открывать эту инструкцию внутри приложения через меню `Помощь` или клавишу `F1`.
14. Видеть номера строк рядом с текстовым редактором.
15. Переходить к узлу из сообщения валидации двойным кликом.
16. Перезагружать текущий файл с диска.
17. Экспортировать текущую модель в отдельный `.yang` файл.
18. Использовать базовую подсветку синтаксиса YANG.
19. Переходить к определению узла в редакторе при выборе узла в дереве.
20. Сохранять текущее раскрытие дерева и выбранный узел после редактирования, добавления узлов и проверки модели.
21. Автоматически выравнивать ручной ввод в редакторе: `Tab` вставляет один уровень отступа, `Enter` продолжает отступ текущей строки.
22. Подсвечивать парную фигурную скобку при установке курсора рядом с `{` или `}`.
23. Использовать выбранную иконку приложения из ресурсов сборки.
24. Автоматически сохранять изменения в открытый `.yang` файл.
25. Искать и заменять текст в редакторе через меню `Правка`.
26. Задавать `config` и `mandatory` через галочки в форме узла.
27. Показывать встроенную инструкцию с базовым Markdown-форматированием.
28. Прокручивать текстовый редактор колесом мыши и полосами прокрутки.
29. Работать с несколькими YANG-файлами через вкладки редактора.
30. Использовать JetBrains Mono в редакторе и менять масштаб текста.
31. Экспортировать минимальный XML-пример по текущей YANG-модели.

## Требования для запуска

Нужен JDK 21.

Для полной проверки YANG-моделей нужна утилита `pyang`.

Установка:

```powershell
python -m pip install -r requirements.txt
```

В Linux-дистрибутив файл `requirements.txt` копируется рядом с `run.sh`.

Если `JAVA_HOME` не задан в системе, перед запуском Maven Wrapper укажите путь к JDK.

Пример для текущего окружения Windows:

```powershell
$env:JAVA_HOME='C:\Program Files\Android\openjdk\jdk-21.0.8'
```

## Запуск приложения

Из корня проекта:

```powershell
.\mvnw.cmd javafx:run
```

## Сборка и проверка

Запуск тестов:

```powershell
.\mvnw.cmd test
```

Сборка jar:

```powershell
.\mvnw.cmd package
```

Сборка Linux-дистрибутива:

```powershell
.\mvnw.cmd -Plinux-dist package
```

Результат Linux-сборки создается в `target/YangModelDesigner-2.0-linux.zip`.

Папка Linux-дистрибутива очищается перед сборкой профиля, чтобы в пакет не попадали устаревшие библиотеки.

Файлы запуска Linux, которые должны попадать в git, находятся в `src/linux`. При сборке профиля `linux-dist` Maven копирует `src/linux/run.sh` в дистрибутив и подставляет актуальное имя jar, модуль и главный класс.

## Основной экран

Главный экран состоит из трех рабочих областей:

1. Слева - дерево YANG-узлов.
2. По центру - текстовый редактор YANG-модели.
3. Справа - свойства выбранного узла и действия над ним.

Внизу находится список сообщений парсинга и валидации.

Иконка окна приложения берется из ресурса `src/main/resources/com/yangdesigner/yangmodeldesigner/app-icon.png`.

Над текстовым редактором находится панель вкладок. Каждая вкладка хранит свой текст, путь файла, состояние изменения и состояние дерева узлов. При переключении вкладок приложение обновляет редактор, дерево и заголовок окна для выбранного файла.

## Встроенная справка

Открыть инструкцию внутри приложения можно через меню:

```text
Помощь -> Инструкция
```

Горячая клавиша:

```text
F1
```

Приложение читает файл `YangDesignerInstruction.md` и показывает его с базовым Markdown-форматированием: заголовки, списки, inline-code и блоки кода. В Linux-дистрибутиве этот файл копируется рядом с `run.sh`.

## Создание новой модели

Используйте меню:

```text
Файл -> Создать
```

Или горячую клавишу:

```text
Ctrl+N
```

Приложение создаст шаблон YANG 1.1-модуля.

Новая модель открывается в отдельной вкладке.

## Открытие YANG-файла

Используйте меню:

```text
Файл -> Открыть...
```

Или горячую клавишу:

```text
Ctrl+O
```

Выберите файл с расширением `.yang`.

После открытия приложение:

1. Загружает текст в редактор.
2. Разбирает модель.
3. Строит дерево узлов.
4. Показывает сообщения об ошибках разбора, если они есть.

Каждый открытый файл загружается в отдельную вкладку. Если файл уже открыт, приложение переключится на существующую вкладку.

Для ручной проверки можно открыть пример:

```text
examples/basic-system.yang
```

## Перезагрузка файла

Перезагрузить текущий файл с диска можно через меню:

```text
Файл -> Перезагрузить
```

Горячая клавиша:

```text
Ctrl+Shift+R
```

Перезагрузка заменяет текущий текст содержимым файла на диске.

## Редактирование текста

Текст YANG-модели редактируется в центральной области.

Слева от редактора отображаются номера строк.

Редактор поддерживает прокрутку колесом мыши и штатными полосами прокрутки.

Редактор использует шрифт JetBrains Mono. Если он не установлен в системе, JavaFX использует резервный моноширинный шрифт.

Масштаб текста редактора меняется через меню:

```text
Вид -> Увеличить масштаб
Вид -> Уменьшить масштаб
Вид -> Сбросить масштаб
```

Горячие клавиши:

```text
Ctrl++
Ctrl+-
Ctrl+0
```

Редактор подсвечивает:

1. YANG statements.
2. Базовые YANG-типы.
3. Строки.
4. Комментарии.
5. Фигурные скобки и `;`.

Если курсор установлен рядом с `{` или `}`, редактор дополнительно подсвечивает эту скобку и найденную парную скобку.

После изменения текста приложение с небольшой задержкой повторно разбирает модель и обновляет дерево узлов.

Отступы в редакторе согласованы с генератором YANG-кода:

1. Один уровень отступа равен двум пробелам.
2. `Tab` вставляет два пробела вместо символа табуляции.
3. `Shift+Tab` уменьшает отступ текущей строки или выделенного блока на один уровень.
4. `Enter` создает новую строку с отступом текущей строки.
5. Если строка перед курсором заканчивается `{`, после `Enter` отступ увеличивается на один уровень.

Если файл изменен и еще не сохранен, в заголовке окна появляется `*`.

Если модель уже связана с файлом, изменения автоматически сохраняются в этот файл с небольшой задержкой. Для новой модели без выбранного файла сначала используйте `Файл -> Сохранить как...`.

Автосохранение работает для активной вкладки, если она уже связана с файлом.

Поиск доступен через меню:

```text
Правка -> Найти...
```

Горячая клавиша:

```text
Ctrl+F
```

Поиск открывается компактной строкой снизу редактора. В ней можно искать вперед и назад, включать учет регистра и закрывать строку кнопкой `Закрыть` или клавишей `Esc`.

Замена доступна через меню:

```text
Правка -> Заменить...
```

Горячая клавиша:

```text
Ctrl+H
```

Замена открывается отдельным окном. В окне замены можно найти следующее совпадение, заменить текущее совпадение или заменить все совпадения. При необходимости включите учет регистра.

## Работа с деревом узлов

Выберите узел в дереве слева.

При выборе узла редактор переводит курсор на строку, где этот узел объявлен в YANG-тексте, и прокручивает редактор так, чтобы определение оказалось ближе к середине экрана.

Рядом с каждым узлом отображается индикатор доступа:

```text
RW - Read-Write
RO - Read-Only
```

`RO` показывается, если у выбранного узла или одного из его родителей задано ограничение:

```text
config: false
```

Если `config: false` не найден, узел считается `RW`.

После редактирования выбранного узла, добавления дочернего узла или запуска проверки приложение пересобирает модель, но сохраняет раскрытые ветки дерева и текущий выбранный узел.

В правой панели отображаются:

1. Имя узла.
2. Тип узла.
3. Путь узла.
4. YANG type.
5. Описание.
6. Ограничения.

Путь строится от корня модели до выбранного узла.

Правая панель разделена на две вкладки:

1. `Текущий узел` - просмотр и редактирование уже выбранного узла.
2. `Новый узел` - создание дочернего узла для выбранного родителя.

## Редактирование свойств через GUI

Откройте вкладку:

```text
Текущий узел
```

Для выбранного узла можно изменить:

1. Имя.
2. YANG type.
3. Описание.
4. Ограничения.

`YANG type` выбирается из выпадающего списка. Поле можно редактировать вручную, если нужен пользовательский typedef или тип с префиксом.

Операторы `config` и `mandatory` задаются отдельными трехсостояниями галочками:

1. Пустое состояние - оператор не будет записан в YANG.
2. Галочка установлена - будет записано `true`.
3. Галочка снята - будет записано `false`.

После изменения нажмите:

```text
Применить
```

Приложение обновит внутреннюю модель и перегенерирует YANG-текст в редакторе.

## Формат ограничений в GUI

Ограничения вводятся построчно в формате:

```text
keyword: value
```

Примеры:

```text
default: localhost
key: name
min-elements: 1
```

Если строка ограничения не содержит `:`, она будет пропущена.

Операторы `config` и `mandatory` не нужно вводить в текстовое поле ограничений: они задаются галочками.

По умолчанию поле ограничений пустое.

## Добавление узлов через GUI

Выберите родительский узел.

Откройте вкладку:

```text
Новый узел
```

Заполните отдельную форму нового узла:

1. Тип нового узла.
2. Имя нового узла.
3. YANG type.
4. Описание.
5. Ограничения.

Нажмите:

```text
Добавить к выбранному узлу
```

Сейчас через GUI можно добавить:

```text
container
list
leaf
leaf-list
choice
case
grouping
uses
typedef
identity
feature
rpc
action
input
output
notification
augment
anydata
anyxml
```

Приложение добавит дочерний узел и перегенерирует YANG-текст.

Для нового `leaf` автоматически задается:

```text
type string;
```

При генерации нового узла substatements вставляются в стабильном порядке:

```text

description
when
must
key
unique
mandatory
default

min-elements
max-elements
ordered-by
presence
if-feature
status
reference
```

При обратной генерации YANG-текста между соседними YANG-узлами вставляется одна пустая строка.

## Экспорт XML-примера

Экспортировать минимальный XML-пример по текущей YANG-модели:

```text
Файл -> Экспортировать XML...
```

Горячая клавиша:

```text
Ctrl+Shift+E
```

XML-пример создается для быстрой визуальной сверки структуры данных:

1. Корневой элемент файла - `<data>`.
2. `namespace` модуля используется как `xmlns` у `<data>`, если он задан.
3. `container` и `list` становятся вложенными XML-элементами.
4. `leaf` и `leaf-list` получают примерные значения по `type`.
5. Если у leaf задан `default`, используется это значение.

Это минимальная генерация примера, а не полноценная генерация XML instance-data со всеми правилами YANG, `choice/case`, `uses`, `augment`, namespaces для нескольких модулей и проверкой значений.

## Проверка модели

Используйте меню:

```text
Инструменты -> Проверить
```

Горячая клавиша:

```text
Ctrl+R
```

Результаты проверки выводятся в нижней панели сообщений.

Проверка выполняется через `pyang`. Приложение пробует запускать `pyang`, затем `python -m pyang`, `python3 -m pyang` и `py -m pyang`.

Если `pyang` не найден, нижняя панель покажет сообщение с командой установки.

Если сообщение связано с конкретным узлом, двойной клик по сообщению выбирает этот узел в дереве. Если `pyang` сообщил номер строки без пути узла, двойной клик переводит редактор на эту строку.

## Текущие проверки

Основная проверка выполняется существующей утилитой `pyang`, а не собственным набором правил приложения.

`pyang` проверяет синтаксис YANG, базовые правила RFC 6020/7950, типы statements и зависимости через `import`/`include`, если нужные модули находятся рядом с открытым файлом или доступны в пути запуска.

## Текущие ограничения MVP

Приложение пока не является полным RFC 7950-редактором.

Ограничения:

1. Парсер поддерживает базовую структуру YANG statements и сохраняет неизвестные statements в AST.
2. Исходное форматирование файла пока не сохраняется полностью при генерации текста.
3. Нет полноценного разрешения `import` и `include` между файлами.
4. Нет строгой проверки всех правил cardinality/order из RFC 7950.
5. Подсветка синтаксиса базовая и пока не является полноценным YANG language server.
6. GUI-редактирование покрывает только часть свойств узла.

## Рекомендованный рабочий процесс

1. Откройте `.yang` файл или создайте новую модель.
2. Проверьте дерево узлов слева.
3. Выберите нужный узел.
4. Измените текст вручную или свойства через правую панель.
5. Если меняли свойства через GUI, нажмите `Применить`.
6. Запустите `Инструменты -> Проверить`.
7. Исправьте ошибки.
8. Сохраните файл.

## Правило обновления инструкции

При каждом изменении функционала нужно обновлять этот файл:

1. Если добавлено новое действие в UI - добавить его в инструкцию.
2. Если изменились горячие клавиши - обновить разделы запуска действий.
3. Если изменились правила валидации - обновить раздел `Текущие проверки`.
4. Если снято или добавлено ограничение MVP - обновить раздел `Текущие ограничения MVP`.
5. Если изменились команды сборки или запуска - обновить разделы запуска и сборки.
